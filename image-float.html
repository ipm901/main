First, user authentication has nothing to do with user.name and user.email git config. And second, compared to the old answer "remove credentials from git", the latest Git for Windows uses Git Credential Manager (Git 2.9.2 => GCM 1.5.0). Check your config (git config -l) and see if "manager" (or, more recently, 2020+: "manager-core") is registered there. If not, set it up with: git config --global credential.helper manager-core Since its v1.3.0 (April 2016), it had a delete command to purge credentials. git credential-manager delete Update 2018: "delete" is now deprecated, use reject: Update 2020:
